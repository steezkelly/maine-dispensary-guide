## 3. DESIGN SYSTEM ASSESSMENT (Full Critique)

### Verdict
Solidly above-average for cannabis industry. Avoids neon-green stoner clichés. Reads as "competent template" rather than "distinctive brand." The tension: it wants to be both **professional authority** (law firm) and **friendly accessibility** (beanbag lobby) — these fight each other.

### Color Palette (Strong)
| Token | Value | Role |
|---|---|---|
| `--color-primary` | `#0D4E50` | Deep teal — distinctive, professional |
| `--color-background` | `#F2F2E2` | Warm bone — avoids clinical white |
| `--color-accent` | `#588157` | Leaf green — functional, not decorative |
| `--color-text` | `#101820` | Maximum contrast navy |
| `--color-soft-green` | `#7A9A6A` | Sage (passes 4.5:1 on white) |

Dark mode is intentional, not an inversion: `#061A1B` bg, `#C4D4B6` primary.

### Typography (Strong)
- **Body:** `Plus Jakarta Sans`, sans-serif — modern, legible
- **Display/Logo:** `Fraunces`, serif — editorial weight, brand personality
- **Pairing:** Works well, gives hierarchy without being flashy

### What Works Well
1. **Grain texture overlay** — `natural-paper.png` at 30% opacity adds physicality
2. **Floating pine tree ornaments** — 12 SVGs at 6% opacity, ambient brand flourish
3. **Dark mode** — fully separate palette, not just inverted
4. **Component consistency** — Callouts, fact boxes, FAQs, disclaimers share one visual language
5. **Accessibility** — Skip links, `prefers-reduced-motion`, semantic landmarks, data table `aria-label`
6. **Sticky header + progress bar** — subtle scroll feedback

### What Weakens the Design
| Issue | Evidence | Severity |
|---|---|---|
| **Border-radius overuse** | `--radius: 1.5rem` (24px) on nav links, dropdowns, cards, tables. Reads as "just discovered border-radius." | 🔶 Medium |
| **44KB Layout.astro = CSS debt bomb** | All site CSS in 1,116 lines of inline `<style>`. No architecture, no scoped components. The esbuild minify warnings (`Unexpected <`, `Unexpected }`) are symptoms. | 🔴 High |
| **Disclaimer box color clash** | `#fff3cd` Bootstrap warning yellow against sophisticated teal/green. Screams legal boilerplate. | 🔶 Medium |
| **Hero imagery weight** | 13MB in `/images/heroes/` (~185KB avg per image). No WebP/AVIF apparent. JPEGs. Likely stock/AI-generated — trust killer if low quality. | 🔴 High (unverified) |
| **Homepage CTA dilution** | "Start Your Application / Calculate Your ROI / Download Founders Bible" buried in text density. Need isolation + visual weight. | 🔶 Medium |
| **Mobile nav pattern** | Full-screen slide-in with centered links = 2018 pattern. `padding-top: 100px` creates thumb-unreachable menus. | 🔶 Low |
| **Grain overlay performance** | `position: fixed` at `z-index: 999` forces compositor repaints every scroll. 30% opacity is not free. | 🔶 Low |
| **Missing micro-interactions** | No button press states, no card hover lift, no link underlines. Static despite progress bar and dark toggle. | 🔶 Low |

### CSS Minify Warnings (Pre-existing, Benign)
Two warnings from esbuild on Layout.astro: `Unexpected "<"` and `Unexpected "}"`.
- **Cause:** Astro compiler emits inline `<style>` with ~44KB CSS. esbuild's minifier sees `>` and `<` characters in CSS selectors (child combinators) and misinterprets them as HTML.
- **Impact:** Zero. Build passes. Runtime is fine.
- **Fix:** Would require splitting CSS into external files or scoped components.
